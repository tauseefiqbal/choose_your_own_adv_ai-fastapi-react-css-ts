# Choose Your Own Adventure AI

An AI-powered interactive story generator. Pick a theme and the app builds a full branching adventure — with multiple paths, choices, and endings — entirely generated by **GPT-4o-mini**. Built with **FastAPI**, **React 19 + TypeScript**, and packaged as a single Docker image for one-command deployment.

## Features

- ✅ Enter any theme (e.g. "pirates", "space station", "medieval fantasy") and get a unique branching story generated by AI
- ✅ Fully interactive narrative — every story node presents you with choices that branch the plot
- ✅ Multiple endings — stories can have winning endings and losing/neutral endings
- ✅ Asynchronous story generation — a job is created instantly and the frontend polls until ready, so the UI never hangs
- ✅ Persistent session tracking — your session is remembered via a cookie so your story history is preserved across page refreshes
- ✅ Single-container deployment — the same Docker image serves both the React frontend and the FastAPI backend
- ✅ SQLite in development, PostgreSQL in production — zero config switching via environment variables
- ✅ End-to-end **TypeScript** frontend with strict type safety
- ✅ Fully documented REST API with Swagger UI at `/docs` and ReDoc at `/redoc`
- ✅ CORS-protected API with configurable allowed origins
- ✅ Restart any story from the beginning at any time

## How to Use the App (Regular User)

### Step 1 — Enter a Theme

Open the app in your browser. You will see a text input with the prompt **"Enter a theme for your interactive story"**.

Type any theme you like — be as creative as you want. Examples:

- `pirates`
- `haunted mansion`
- `space station rescue`
- `medieval dragon quest`
- `underwater kingdom`

Then click **Generate Story**.

### Step 2 — Wait for Generation

The app will display a loading indicator while GPT-4o-mini generates your unique branching story. This typically takes **10–20 seconds**. The frontend polls the server automatically — you do not need to refresh.

### Step 3 — Make Choices

Once ready, the first scene of your story appears. At the bottom you will see **2 or more choice buttons**. Click any button to follow that path — the story advances to the next scene based on your choice.

Every choice you make shapes the plot. Different paths lead to completely different scenes and outcomes.

### Step 4 — Reach an Ending

Eventually you will reach an ending node. There are two types:

- **Winning ending** — displayed with a **"Congratulations"** message.
- **Neutral / losing ending** — displayed with **"The End"**.

After any ending you can click **Restart Story** to start the same story over, or **New Story** to generate a brand-new adventure with a different theme.

## Environment Variables

### Backend

Place these in `backend/.env` for local development. In production they are injected by Render.

| Variable          | Required   | Default | Description                                                              |
| ----------------- | ---------- | ------- | ------------------------------------------------------------------------ |
| `OPENAI_API_KEY`  | **Yes**    | —       | Your OpenAI API key (get one at platform.openai.com/api-keys)            |
| `DEBUG`           | No         | `False` | Set to `True` for local dev (enables SQLite)                             |
| `DATABASE_URL`    | No         | —       | Full SQLite URL when `DEBUG=True`, e.g. `sqlite:///./database.db`        |
| `API_PREFIX`      | No         | `/api`  | URL prefix for all API routes                                            |
| `ALLOWED_ORIGINS` | No         | `""`    | Comma-separated list of allowed CORS origins                             |
| `DB_HOST`         | Prod only  | —       | PostgreSQL host (auto-wired by Render)                                   |
| `DB_PORT`         | Prod only  | —       | PostgreSQL port (auto-wired by Render)                                   |
| `DB_NAME`         | Prod only  | —       | PostgreSQL database name (auto-wired by Render)                          |
| `DB_USER`         | Prod only  | —       | PostgreSQL user (auto-wired by Render)                                   |
| `DB_PASSWORD`     | Prod only  | —       | PostgreSQL password (auto-wired by Render)                               |

### Frontend

Place these in `frontend/.env` for local development.

| Variable     | Required | Default | Description                                                           |
| ------------ | -------- | ------- | --------------------------------------------------------------------- |
| `VITE_DEBUG` | No       | —       | Set to `true` to enable Vite proxy from `/api` → `http://localhost:8000` |

## Local Development Setup

### Prerequisites

- Python 3.13+
- Node.js 20+
- An OpenAI API key ([platform.openai.com/api-keys](https://platform.openai.com/api-keys))

### Clone the Repository

```bash
git clone https://github.com/tauseefiqbal/choose_your_own_adv_ai-fastapi-react-css-ts.git
cd choose_your_own_adv_ai-fastapi-react-css-ts
```

### Backend Setup

```bash
# 1. Navigate to the backend folder
cd backend

# 2. Create and activate a virtual environment
python -m venv venv
venv\Scripts\activate        # Windows
# source venv/bin/activate   # macOS / Linux

# 3. Install dependencies
pip install -r requirements.txt

# 4. Create the .env file
cp .env.example .env         # or create it manually (see Environment Variables above)
# Edit backend/.env and set OPENAI_API_KEY=sk-...

# 5. Start the backend (runs on http://localhost:8000)
uvicorn main:app --reload --port 8000
```

Swagger docs are available at [http://localhost:8000/docs](http://localhost:8000/docs).

### Frontend Setup

```bash
# 1. Navigate to the frontend folder
cd frontend

# 2. Install dependencies
npm install

# 3. Create the .env file
echo VITE_DEBUG=true > .env

# 4. Start the dev server (runs on http://localhost:5173)
npm run dev

# 5. Type-check + production build
npm run build
```

The Vite dev server proxies `/api` requests to the FastAPI backend on port 8000 when `VITE_DEBUG=true`.

## API Reference

All endpoints are prefixed with `API_PREFIX` (default `/api`). Full interactive docs at `/docs`.

### POST /api/stories/create

Kicks off an asynchronous story generation job.

**Request body**

```json
{ "theme": "pirates" }
```

**Response**

```json
{ "job_id": "abc123", "status": "pending" }
```

### GET /api/jobs/{job_id}

Polls the status of a generation job.

**Response**

```json
{
  "job_id": "abc123",
  "status": "completed",
  "story_id": 42,
  "error": null
}
```

`status` is one of: `pending`, `processing`, `completed`, `failed`.

### GET /api/stories/{story_id}/complete

Returns a fully expanded story tree, ready to render.

**Response (abridged)**

```json
{
  "id": 42,
  "title": "The Pirate's Curse",
  "session_id": "…",
  "created_at": "2026-04-24T12:00:00Z",
  "root_node": { "id": 1, "content": "…", "is_ending": false, "options": [ … ] },
  "all_nodes": {
    "1": { "id": 1, "content": "…", "options": [ { "text": "Climb the mast", "node_id": 2 } ] },
    "2": { "id": 2, "content": "…", "options": [ … ] }
  }
}
```
